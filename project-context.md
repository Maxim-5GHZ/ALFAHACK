# Контекст проекта: «Альфа.Старт» (Alfa.Start)

## Описание проекта

«Альфа.Старт» — это веб-приложение (MVP) для молодых предпринимателей (17–25 лет). Сервис представляет собой интерактивного помощника на базе ИИ, который помогает выбрать нишу, рассчитывает первичный бизнес-план (окупаемость, расходы) и предлагает открыть ИП и расчетный счет в Альфа-Банке.

## Структура репозитория

Проект разделен на две независимые части (монорепозиторий):

- `/frontend` — клиентская часть (Next.js, TypeScript).
- `/backend` — серверная часть и логика ИИ (Python, FastAPI).

---

## 1. Frontend (Клиентская часть)

### Стек технологий:

- Фреймворк: Next.js (App Router, версия 14+).
- Язык: TypeScript (строгая типизация обязательна).
- Стилизация: Tailwind CSS.
- Компоненты UI: shadcn/ui (приоритет использования) или Radix UI.
- Иконки: Lucide React.
- Формы: react-hook-form + zod (валидация).
- HTTP-клиент: fetch (обёртки в `lib/api.ts`).
- Аутентификация: JWT, токен хранится в `localStorage` (ключ `token`).

### Дизайн-система (Стиль Альфа-Банка):

Интерфейс должен быть современным, минималистичным, ориентированным на молодежь.
**Цветовая палитра:**

- Главный красный (Primary): `#EF3124` (использовать для главных CTA-кнопок).
- Черный (Текст/Заголовки): `#0B1F35` или `#000000`.
- Белый (Фон): `#FFFFFF`.
- Светло-серый (Фон карточек/инпутов): `#F3F4F6`.
- Акцентный зеленый (успех/окупаемость): `#10B981`.

**Интерактивность UI-компонентов:**

- **Button** (`components/ui/button.tsx`): три варианта (`default`, `outline`, `ghost`), три размера (`default`, `lg`, `icon`). Нажатие: `active:scale-[0.97]`. Ховер: поднятие тени (`hover:shadow-md`). Фокус: кольцо `ring-primary/50`. Отключённое состояние: `opacity-50`, без анимации нажатия.
- **Input** (`components/ui/input.tsx`): фокус — `ring-2 ring-primary`; поддержка `showToggle` для пароля (иконка Eye/EyeOff).
- **Feature cards** (главная): ховер — подъём `-translate-y-0.5`, тонкая обводка `border-primary/20`, иконка масштабируется `scale-110`.
- **Навигация** (Header): активная ссылка подсвечивается `text-primary`, ховер — плавный переход цвета.

**Правила UI/UX:**

- **Mobile First**: вся вёрстка строится от мобильного разрешения (базовые стили для `xs`/`sm`) с последующим расширением через `md:` / `lg:`. Хедер — `h-14` на мобилках, `md:h-16`. Сайдбар — оверлей на мобилках, фиксированный на `md:`.
- Избегать скучных длинных текстовых форм. Использовать интерактивные карточки для выбора ниши и ползунки (sliders) для выбора бюджета.
- Обязательно наличие экрана загрузки (скелетоны или красивый спиннер), пока бэкенд опрашивает ИИ.
- Финальный экран должен содержать выделенные блоки: "Смета", "План действий" и **Красные кнопки интеграции** ("Открыть ИП бесплатно", "Подобрать эквайринг").
- Все кликабельные элементы должны иметь визуальную обратную связь (hover/active/focus).

## 2. Backend (Серверная часть)

### Стек технологий:

- Фреймворк: FastAPI (Python 3.10+).
- Сервер: Uvicorn.
- Валидация данных: Pydantic.
- База данных: PostgreSQL (async) через SQLAlchemy 2.0 + asyncpg.
- ORM: SQLAlchemy (DeclarativeBase), асинхронные сессии.
- Аутентификация: JWT (python-jose) + bcrypt (passlib).
- Запросы к внешним API (LLM/Geo): `httpx`.

### Архитектура и правила:

- Настроить CORS middleware для приема запросов из `/frontend`.
- Точка входа: `main.py`.
- Роутинг: Вынести логику API в `/api/routes`.

### Auth (JWT):

- **Config**: `AUTH_SECRET_KEY`, `JWT_ALGORITHM` (HS256), `ACCESS_TOKEN_EXPIRE_MINUTES` (1440) — в `core/config.py` / `.env`.
- **Security**: `core/security.py` — `hash_password`, `verify_password` (bcrypt), `create_access_token`, `decode_access_token` (python-jose).
- **Endpoints** (префикс `/api/v1/auth`):
  | Метод | Путь | Описание |
  |---|---|---|
  | POST | `/api/v1/auth/register` | Регистрация (email, username, password) → `{access_token}` |
  | POST | `/api/v1/auth/login` | Вход (email, password) → `{access_token}` |
  | GET | `/api/v1/auth/me` | Текущий пользователь (требует Bearer token) |

- **User model** (`models/user.py`): `id`, `email` (unique), `username`, `hashed_password`, `created_at`.
- **Dependency**: `get_current_user` (извлекает и верифицирует JWT из заголовка Authorization).
- Взаимодействие с ИИ: Бэкенд должен формировать промпт на основе данных от пользователя, отправлять его в LLM (например, через OpenAI API формат) и **ОБЯЗАТЕЛЬНО** требовать ответ в формате жестко структурированного JSON.
